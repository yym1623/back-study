# CS(컴퓨터공학) 학습 표준 순서와 그 이유

## 전체 순서

```
1. 프로그래밍 기초
2. 자료구조
3. 알고리즘
4. 컴퓨터 구조 (Computer Architecture)
5. 운영체제 (Operating System)
6. 네트워크 (Computer Network)
7. 데이터베이스 (Database)
8. (심화, 선택) 컴파일러 / 분산시스템 / 보안 등
```

이 순서는 한국/해외 컴공 학과에서 일반적으로 채택하는 표준 커리큘럼 순서다. 각 단계는 이전 단계의 개념을 전제로 하기 때문에 이 순서로 배우는 것이 자연스럽다.

---

## 1. 프로그래밍 기초
- 문법, 조건문, 반복문, 함수, 포인터(C 기준) 등
- **먼저 배우는 이유**: 이후 모든 과목이 "코드로 구현"하는 것을 전제로 하기 때문에 반드시 선행되어야 함

## 2. 자료구조 (Data Structures)
```
배열 → 연결리스트 → 스택/큐 → 트리 → 힙 → 해시테이블 → 그래프
```
- 데이터를 "어떻게 저장할지"를 다루는 영역
- **이 다음 알고리즘을 배우는 이유**: 알고리즘은 결국 자료구조 위에서 동작하는 처리 로직이기 때문에, 자료구조를 먼저 알아야 알고리즘이 이해됨

## 3. 알고리즘 (Algorithms)
```
시간복잡도(Big-O) → 정렬 → 탐색 → 재귀 →
분할정복 → 그리디 → 동적계획법(DP) → 그래프 알고리즘(DFS/BFS/최단경로)
```
- 자료구조에 저장된 데이터를 "어떻게 효율적으로 처리할지"를 다루는 영역

## 4. 컴퓨터 구조 (Computer Architecture)
```
논리 회로 → CPU 구성(ALU, 레지스터) → 명령어 사이클 →
메모리 계층구조(캐시, RAM) → 버스 → I/O
```
- 하드웨어가 실제로 어떻게 구성되어 있는지를 다루는 영역
- **운영체제보다 먼저 배우는 이유**: 운영체제는 결국 하드웨어를 관리하는 소프트웨어이기 때문에, 하드웨어 구조를 먼저 알아야 운영체제가 "왜 이런 방식으로 관리하는지" 이해가 됨

## 5. 운영체제 (Operating System)
```
프로세스 vs 스레드 → CPU 스케줄링 →
동기화(뮤텍스, 세마포어) → 데드락 →
메모리 관리(가상메모리, 페이징) → 파일시스템
```
- 하드웨어(컴퓨터 구조)를 소프트웨어가 어떻게 관리하고 추상화하는지를 다루는 영역
- **네트워크보다 먼저 배우는 이유**: 네트워크 통신도 결국 "프로세스 간의 통신"이며, 소켓 프로그래밍을 이해하려면 운영체제의 프로세스 개념이 먼저 필요함

## 6. 네트워크 (Computer Network)
```
OSI 7계층 / TCP-IP 4계층 모델
   ↓
물리계층 → 데이터링크(이더넷) →
네트워크계층(IP, 라우팅) → 전송계층(TCP/UDP) →
응용계층(HTTP, DNS, FTP)
```
- 컴퓨터끼리 어떻게 통신하는지를 다루는 영역
- IP 주소, 포트, NAT, 포트포워딩, TCP/UDP, HTTP, DNS 등이 이 영역에 속함

## 7. 데이터베이스 (Database)
```
관계형 모델 → SQL → 정규화 →
트랜잭션(ACID) → 인덱스 → 동시성 제어
```
- 데이터를 영구적으로 저장하고 관리하는 방법을 다루는 영역
- **운영체제와 네트워크 다음에 배우는 이유**: DB의 동시성 제어는 운영체제의 동기화 개념과 연결되고, 클라이언트-서버 간 DB 접근은 네트워크 통신 개념과 연결되기 때문

## 8. 심화 영역 (선택)
```
컴파일러 (프로그래밍 언어가 어떻게 기계어로 변환되는지)
분산시스템 (여러 서버가 어떻게 협업하는지 — MSA, 로드밸런싱 등)
보안 (암호학, 네트워크 보안)
```

## 요약 표

| 순서 | 과목 | 핵심 질문 | 선행 과목이 필요한 이유 |
|---|---|---|---|
| 1 | 프로그래밍 기초 | 코드를 어떻게 짜나 | (최초 선행 지식) |
| 2 | 자료구조 | 데이터를 어떻게 저장하나 | 코드로 직접 구현하며 배우기 때문에 프로그래밍 기초 필요 |
| 3 | 알고리즘 | 데이터를 어떻게 효율적으로 처리하나 | 알고리즘은 자료구조 위에서 동작 |
| 4 | 컴퓨터 구조 | 하드웨어는 어떻게 생겼나 | (독립적으로 배울 수 있으나 운영체제 이전 배치가 자연스러움) |
| 5 | 운영체제 | 소프트웨어가 하드웨어를 어떻게 관리하나 | 컴퓨터 구조(하드웨어)를 알아야 관리 방식 이해 가능 |
| 6 | 네트워크 | 컴퓨터끼리 어떻게 통신하나 | 프로세스/소켓 개념(운영체제)이 선행되어야 이해 쉬움 |
| 7 | 데이터베이스 | 데이터를 어떻게 영구 저장/관리하나 | 동시성 제어(운영체제) + 클라이언트-서버 통신(네트워크) 개념 필요 |
