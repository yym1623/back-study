1. require('http')는 어떻게 되는 거냐
Node.js는 설치될 때 **내장 모듈(built-in modules)**을 같이 갖고 있어요. http, fs, path, crypto 등이 전부 Node 실행 파일 안에 이미 포함되어 있는 코드예요.
jsconst http = require('http');
이건 "외부에서 뭘 다운받아오는 게" 아니라, Node 런타임 안에 이미 내장되어 있는 http 라는 이름의 모듈을 가져오는 것이에요. npm install 필요 없음. Node 자체가 서버 만들 수 있는 능력을 처음부터 갖고 있는 거예요 (이게 Node가 애초에 "서버사이드 JS 런타임"으로 만들어진 이유이기도 해요).
http 모듈 안에는 createServer 같은 함수들이 미리 구현되어 있어요.
2. http.createServer(콜백함수) — 서버 "설계도" 만들기
jsconst server = http.createServer((req, res) => {
  // 요청 올 때마다 이 함수가 실행됨
});
이 시점에서 하는 일: "요청이 오면 이 함수를 실행해라"라는 규칙을 정의한 서버 객체(server)를 만드는 것.
중요한 건 이 시점엔 아직 서버가 실제로 켜진 게 아니에요. 그냥 "이런 로직으로 동작할 서버"라는 청사진만 만든 상태.
(req, res) => {...} 이 콜백 함수는 Node가 자동으로 호출해주는 함수인데:

req (request): 클라이언트가 보낸 요청 정보 (URL, method, headers, body 등)가 담긴 객체
res (response): 클라이언트한테 응답을 보낼 때 쓰는 객체 (res.write(), res.end() 등의 메서드 보유)

3. server.listen(3000, ...) — 실제로 서버 켜기 (포트 열기)
jsserver.listen(3000, () => {
  console.log('서버 실행 중: http://localhost:3000');
});
여기서 진짜 일이 일어나요: OS(운영체제)에게 "3000번 포트로 들어오는 네트워크 요청을 나(이 프로세스)한테 넘겨줘"라고 요청하는 거예요.

컴퓨터에는 수많은 포트(0~65535)가 있고, 각 포트는 특정 프로그램이 "점유"할 수 있음
listen(3000)을 호출하면 Node 프로세스가 3000번 포트를 점유하고, 그 포트로 들어오는 요청을 계속 감시(listen)하기 시작함
이 순간부터 프로세스가 종료되지 않고 계속 살아있음 (이게 서버가 "떠있다"는 상태)

4. 요청이 들어오면 무슨 일이 일어나냐 (전체 흐름)
① 브라우저에서 http://localhost:3000 접속
      ↓
② OS가 "3000번 포트로 요청 왔다" → Node 프로세스에 전달
      ↓
③ Node가 이 요청을 http.createServer에 정의해둔 콜백함수 (req, res) => {...} 실행
      ↓
④ 콜백 안에서 req로 요청 내용 확인, res로 응답 작성
      ↓
⑤ res.end()가 호출되면 → 응답이 클라이언트(브라우저)로 전송됨
      ↓
⑥ 브라우저 화면에 "Hello Node!" 표시
      ↓
⑦ Node 프로세스는 안 죽고 계속 다음 요청 기다림 (listen 상태 유지)
